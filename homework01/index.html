Перелік основних команд, якi потрібно записати:

git config --global user.name blabla - команда яка вказує ім'я юзера після встановлення гіт;

git config --global user.email blabal@bla - команда яка вказує емейл юзера після встановлення гіт;

git init - процес Гіт ініціалізації проєкту, при якому буде створена папка з ім'ям .git всередині якої будуть знаходитися необхідні файли репозиторію;

git status - команда для демонстрації наявності ініціалізації гіта у проєкті, стану файлів з приводу їх змін, також показує на якій вітці ви знаходитесь;

git add - додавання локального файлу до версійного контролю;

git commit - фіксація змін у файлах з можливістю додавання коментарів;

git log - перечисляє усі коміти, які були зроблені в репозиторії від новіших до старіших; має різноманітні опції для пошуку комітів;

git checkout - створення нової вітки, перемикання між існуючими вітками;

git branch - вказує на список існуючих віток та має опції фільтрації таких; показує останній коміт на кожній вітці; удаляє вітку; змінює локально її ім'я;

git push - використовується для зв'язку з віддаленим репозиторієм, аналізу змін, які відсутні у такому і передача їх. Під час використання присутній процес аутентифікації;

git pull - забирає зміни з наявного віддаленого репозиторію і потім зливає її з батьківською віткою;

git merge - злиття віток;

git clone - клонування віддаленого репозиторію у каталог файлів комп'ютера з можливістю вказати ім'я папки у яку він буде перенесений.

Написати що таке .gitignore файл:

.gitignore - приховування файлів чи папок, котрі не повинні бути перенесені у віддалений репозиторій.

Advanced:

git revert - видаляє зміни після коміту, але залишає сам коміт у історії;

git reset - видалення коміту і зміни, котрі були внесені після такого\ видалення коміту без видалення змін, котрі були внесені після такого;

git rebase - перебазування віток: перенесення комітів з однієї гілки у іншу гілку, має менше шагів ніж merge;

git cherry-pick - надає змогу перенесення одного коміта між гілками;

git stash - відкладення змін у робочій копії та можливiсть створення такого коміту в інший час.
